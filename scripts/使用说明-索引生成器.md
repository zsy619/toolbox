# 目录索引页面生成器 - 使用说明

## 概述

自动为项目所有子目录生成 `index.html` 导航页面,实现文件浏览和目录导航功能。

---

## 🚀 快速开始

### 方法1: Bash脚本 (macOS/Linux 推荐)

```bash
cd /Volumes/E/JYW/工具箱
./scripts/generate-index-pages.sh
```

### 方法2: Python脚本 (跨平台)

```bash
cd /Volumes/E/JYW/工具箱
python3 scripts/generate-index-pages.py
```

---

## ✨ 核心特性

### 1. 自动适应任意深度的目录结构

脚本会自动:
- 计算目录相对路径
- 根据当前深度生成正确的资源引用
- 支持无限层级嵌套

示例:
```
工具箱/
├── apps/                    # 资源路径: ../assets/
│   ├── lottery/             # 资源路径: ../../assets/
│   │   └── 3d_smart.html  # 资源路径: ../../../assets/
│   └── games/              # 资源路径: ../../assets/
│       └── 2048/          # 资源路径: ../../../assets/
│           └── index.html    # 资源路径: ../../../../assets/
```

### 2. 动态生成相对路径引用

根据目录深度自动调整:
```javascript
// apps/ 目录 (深度1)
<link href="../assets/css/layui.min.css">

// apps/lottery/ 目录 (深度2)
<link href="../../assets/css/layui.min.css">

// apps/games/2048/ 目录 (深度3)
<link href="../../../assets/css/layui.min.css">
```

### 3. 保持页面样式和功能的一致性

统一的设计风格:
- 渐变色头部
- 卡片式布局
- 鼠标悬停动画
- 响应式网格布局

### 4. 正确处理特殊字符和空格的文件名

使用URL编码:
```python
# 文件名: 测试 文件.html
# 编码后: %E6%B5%8B%E8%AF%95%20%E6%96%87%E4%BB%B6.html
<a href="%E6%B5%8B%E8%AF%95%20%E6%96%87%E4%BB%B6.html">测试 文件.html</a>
```

### 5. 提供清晰的目录导航界面

功能包含:
- 📁 目录列表(带图标)
- 📄 文件列表(带类型图标)
- 🔍 实时搜索过滤
- 🍞 面包屑导航
- ⬆️ 返回上级链接

## ⚠️ 注意事项

### 1. 覆盖现有index.html

脚本会覆盖同名的 `index.html` 文件,建议:
- 备份重要的index.html
- 或将配置目录加入排除列表

### 2. 资源文件依赖

生成的页面需要以下资源:
```
assets/
├── css/
│   ├── layui.min.css
│   ├── fontawesome.min.css
│   └── style.css
└── js/
    └── layui.min.js
```

### 3. 执行权限

Bash脚本需要执行权限:
```bash
chmod +x scripts/generate-index-pages.sh
```

## 🐛 故障排查

### 问题1: 样式加载失败

**现象**: 页面显示无样式
**原因**: 资源路径错误
**解决**:
```bash
# 检查assets目录存在
ls -la assets/css/
ls -la assets/js/
```

### 问题2: 无法打开文件

**现象**: 点击文件提示404
**原因**: 特殊字符未编码
**解决**: 使用Python版本,已内置URL编码

### 问题3: 中文文件名乱码

**现象**: 中文文件名显示为乱码
**原因**: 文件编码问题
**解决**:
```bash
# 检查文件编码
file -I filename.html

# 使用Python版本,支持UTF-8
python3 scripts/generate-index-pages.py
```
