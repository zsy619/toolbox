# 工具箱项目结构优化方案

## 📊 当前问题诊断

### 1. 结构混乱
- 根目录42个文件,缺乏逻辑分类
- HTML工具、脚本、文档、配置混杂
- 临时测试文件未清理

### 2. 冗余文件
- games目录: 大量 `.backup` 文件(100+个)
- images目录: 多版本重复功能文件
- 根目录: 验证文件、临时测试页面

### 3. 命名不规范
- 中英文混用(如"图片合并-05.md")
- 缺少统一版本号规则
- 部分文件名称不直观

### 4. 文档分散
- 相关文档散落在各处
- 缺少统一的README说明
- 历史版本堆积

## 📋 执行步骤

### 第一阶段: 备份与准备
```bash
# 1. 创建备份分支
git checkout -b backup-before-organization

# 2. 创建临时备份目录
mkdir -p archive/temp-backup
cp -r * archive/temp-backup/
```

### 第二阶段: 创建新目录结构
```bash
mkdir -p apps/{lottery,ai-tools,image-processing,games,utilities,multimedia,layouts,demos,tests}
mkdir -p scripts docs/{guides,image-guides,page-docs} config archive/{old-versions,temp-files,backups}
```

### 第三阶段: 文件迁移
```bash
# 批量移动脚本
mv *.sh *.py scripts/

# 移动配置文件
mv robots.txt sitemap.xml *.xml *.html verify* config/

# 迁移福彩工具
mv fucai/* apps/lottery/

# 迁移AI工具
mv ai/* apps/ai-tools/

# 迁移图像工具
mv images/*.html apps/image-processing/
mv images/*.png assets/images/

# 迁移实用工具
mv compound_interest_calculator.html money.html random_generator.html schedule.html uuid.html no-sequence.html apps/utilities/

# 迁移多媒体工具
mv hls.html ws_stream.html ws_websocket.html apps/multimedia/

# 迁移布局工具
mv *布局*.html apps/layouts/
```

### 第四阶段: 清理冗余
```bash
# 删除备份文件
find apps/games -name "*.backup" -delete

# 删除重复图像工具文件
rm apps/image-processing/image_merge\ copy.html
rm apps/image-processing/image_merge_complete.html
rm apps/image-processing/layui_image_merge_final.html
rm apps/image-processing/image_splitter\ copy.html

# 移动测试页面
mv test-wechat-share.html temp_og_image.html verify-meta-tags.html apps/tests/

# 归档历史文档
mkdir -p docs/image-guides/图形合并历史
mv images/图片合并-*.md docs/image-guides/图形合并历史/
```

### 第五阶段: 文档整理
```bash
# 移动指南文档
mv 微信分享*.md 快速测试指南.md 修复总结.md js-standards.md docs/guides/

# 移动页面文档
mv *.md docs/page-docs/
# 注意: ORGANIZATION.md 和 README.md 保留在根目录
```

### 第六阶段: 创建README文件
为每个主要目录创建 README.md 说明文档

## 📝 命名规范

### 目录命名
- 采用英文 kebab-case: `image-processing`, `ai-tools`
- 功能明确,不超过3层深度

### 文件命名
- HTML工具: `工具功能.html` (如 `compound_interest_calculator.html`)
- 文档: `日期_主题_版本.md` (如 `20250120_整理方案_v1.0.md`)
- 脚本: `功能.扩展名` (如 `cleanup_backups.py`)

### 历史文档
- 归档到 `archive/` 或相应目录的 `history/` 子目录
- 保留原始名称,添加说明
