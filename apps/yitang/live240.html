拆、推、评、算 JTBD思维 需求分析 任务地图

"在彻底理解'问题空间'之前跳进'解决方案空间'是创业的大忌"

一、核心理念

JT

JTBD思维

需求不是关于产品本身，而是客户在特定环境下想要达成的"进步"。

  • 核心逻辑：客户"雇佣"产品来完成某项任务
  • 关键启发：真实需求隐藏在客户的挫败感和未完成的任务中

需求基础公式

需求 = 用户 × 场景 × 问题

用户

具备属性和支付/决策能力的载体

场景

触发需求的特定环境、时间与状态

问题

现实与预期的差距，是需求的"原动力"

二、需求分析四字口诀

拆

拆解

训练拆解用户需求的意识（基本功）

推

推演

训练推演用户使用场景，理解问题

评

评估

训练定性评估需求可行性与价值

算

计算

训练定量估算市场规模，理性决策

需求分析爬山地图（能力段位）

段位 核心能力 关键表现
L1 模糊理解 只谈初心、想服务所有人 开始有意识拆解
L2 主动拆解 能细拆用户、识别差异性、剥离非用户 开始多工具专业分析
L3 定性可行性 推演场景、三维评估、专业取舍 开始全局战略思考
L4 定量天花板 有效对标、估算规模、交叉验证 动态调整与预判
L5 全局决策 需求与产品、商业、增长匹配 战略匹配能力
L6 动态预测 预判规模、竞争与市场变化 长期视角与调整

三、拆解工具与框架

用户拆解清单

2C用户维度

人口统计、角色阶段、能力状态、行为习惯、价值观念

2B用户维度

业务类型、规模性质、标准程度、决策文化、财务特征

场景拆解清单

2C场景维度

时间周期、空间环境、身心状态、特别时刻、稀缺窗口

2B场景维度

流程环节、操作地点、人物协作、时间节点、重大事件

问题拆解清单

2C问题维度

上层功能、深度体验、上层社交、深度情感、转化成钱

2B问题维度

业务问题、技术问题、管理问题、战略问题、风险管控

任务洞察工具

1. 圈定核心执行者

产品不是卖给"人口标签"，而是卖给"正在挣扎中、渴望改变的人"。

核心执行者是任务的主角，所有洞察围绕其展开。

2. 圈定核心任务

  • 愿景/抱负 → 大任务 → 小任务 → 微任务
  • 选择"抽象程度适中"的层级（一般在2-3之间）

3. 任务描述公式

[动词] + [对象] + [情境]
动词

需主动、结果导向

对象

不含产品名称

情境

限定触发边界

4. 完整任务地图（8阶段）

1. 定义
制定计划，确立目标
2. 定位
寻找工具、信息或资源
3. 准备
摆放资源，调整环境
4. 确认
再次检查，确保万无一失
5. 执行
真正的核心动作
6. 监测
观察执行是否顺畅
7. 调整
补救和调整
8. 结束
收尾、清洗、整理、复盘

5. 机会筛选矩阵

核心机会（重点做）

高重要性 + 低满意度

雪中送炭，如打车前的"不知道车多久来"

竞争红海（谨慎做）

高重要性 + 高满意度

锦上添花，除非有核心壁垒

过度服务（可砍掉）

低重要性 + 高满意度

可能有颠覆式创新机会

机会渺茫（尽量不做）

低重要性 + 低满意度

用户虽然吐槽，但其实不在乎

四、隐藏洞察挖掘

四种力量

推力 + 拉力 > 焦虑 + 习惯

推力

现状的痛苦

拉力

新方案的吸引力

焦虑

对未知的恐惧

习惯

对旧方式的依赖

三种任务

溢价空间来源

功能任务

解决具体操作（基础）

情感任务

满足心理感受（溢价关键）

社交任务

塑造社交形象（身份认同）

期望成果公式

[变化方向] + [衡量单位] + [核心对象] + [相关说明]

例如："最小化收集文档所需的时间，如工资单、收据等"

在关键瞬间植入"让用户感知到价值"的细微信号，如声音、触感、视觉反馈等，决定产品的"手感"与信任感。

五、实战示例：隔代育儿场景

公司E的洞察

核心任务

在无法亲自看护时，确保孩子获得连贯且高质量的家庭教育。

三种任务挖掘

功能任务

远程控制教育一致性

情感任务

减轻焦虑与内疚

社交任务

维持孝道与家庭和谐

公司F的四个机会方向

1. 情景化沟通话术生成器

将问责转化为协作，将"教育"姿态转化为"求助"姿态

父母输入具体问题、老人性格标签，生成以"感谢"开头、以"求助"结尾的对话脚本

2. 祖辈成就"放大器"

让老人的付出与进步"被看见"，将隐性劳动转化为家庭认可的显性价值

"家庭育儿高光时刻"共享相册，父母上传孩子好表现并定向感谢，老人可一键分享到社交圈

3. 孩子行为"翻译器"与家庭勋章

将孩子的行为数据转化为全家共同解读的"故事"，并将集体进步游戏化

"家庭专属的宝宝行为周报"生成器，记录正面/挑战行为，生成报告并推荐全家合力完成的小挑战

4. 祖辈"育儿盟友"社交圈

将老人从孤立的家庭育儿执行者，嵌入具有新方法示范效应的同辈社群中

基于小区或广场舞群的"祖辈育儿妙招快闪群"，由活跃奶奶分享"真管用"的小技巧，发起线下讨论

总结

需求分析是一个从模糊到清晰、从定性到定量、从现象到本质的系统工程。掌握"拆、推、评、算"四步法，配合JTBD思维与各类工具，可系统识别真实需求、评估机会价值，并为产品创新提供扎实的洞察基础。
